**Parent plan:** `docs/superpowers/v5-v6-planning-preview.md` § v5.0 Production Hardening And Distribution -> observability+auditability theme (2nd cycle, 1st sub-sprint, after D-87 in-memory seed, D-88 tool-loop integration, D-89 file-backed persistence, D-90 readAuditLog query side).

**Branch:** `feature/d36-gate2-live` (current). This sub-sprint is committed on top of D-101 (`5aa3181`).

**Goal:** Add a `dumpAuditLog` function that reads a JSONL audit log via the existing D-90 `readAuditLog` query side and renders a human-or-machine-friendly dump. The function answers: "I have a persisted audit log file; show me its contents in a form I can read in a terminal OR pipe to `jq` for downstream tooling."

The dump surfaces three classes of metadata:

1. **File metadata**: file path, total events found, filter summary.
2. **Per-event rendering** (text format): index, ISO timestamp, event kind, JSON-encoded payload.
3. **Per-event rendering** (json format): all events as a JSON array under a `events` key, with surrounding metadata.

This is a **same-theme deepening** sub-sprint (NOT a cross-theme bridge). D-87 gave us the in-memory seed; D-88 wired it into the tool loop; D-89 made it survive process restarts; D-90 added the read side. D-102 ties the lifecycle together by producing a narrative ("here is what my agent has been doing") that complements D-90 ("here are the raw bytes"). A future user-facing `deepwhale audit tail` command (D-103+ candidate) can wrap `dumpAuditLog` to print the result to stdout.

The output is **string-based** (text or JSON) but the function also returns the events it rendered (`AuditDumpResult.events`). This keeps the function observable for tests and lets CLI wrappers stream the events separately if needed.

For testability, the function is async (because `readAuditLog` is async) but does not write to stdout itself. It returns a `Promise<AuditDumpResult>` that tests and future CLI commands can inspect or `process.stdout.write`. The function does not log to the AuditLog (would be infinite recursion).

   2. Apply `eventKinds` filter: keep only events whose `kind` is in the filter set.
   3. Apply `sinceTimestamp` filter: keep only events whose `timestamp >= sinceTimestamp`.
   4. Format dispatch:
      - `text`: header line + per-event lines `[NNN] ISO-8601-timestamp kind { payload }`. Sort by timestamp ascending.
      - `json`: `{ filePath, format, totalEvents, events }` (pretty-printed with 2-space indent).
   5. Return `AuditDumpResult` with both the filtered events and the rendered dump.

```ts
/**
 * dumpAuditLog -- D-102 v5.0 observability+auditability 2nd cycle.
 *
 * Reads a JSONL audit log file (via the D-90 readAuditLog query
 * side) and renders a human-or-machine-friendly dump. Together
 * with D-87 (in-memory seed), D-88 (tool-loop integration),
 * D-89 (file-backed persistence), and D-90 (read side), D-102
 * completes the v5 observability+auditability 2nd cycle: the
 * 5th and final piece of the audit-log lifecycle (read + render).
 *
 * The function does NOT write to stdout; it returns a
 * Promise<AuditDumpResult> containing both the filtered events
 * AND the rendered dump string. Future CLI wrappers can stream
 * the dump to process.stdout.
 *
 * Supports two formats:
 *   - 'text' (default): header + per-event lines with ISO timestamps
 *     and JSON-encoded payloads; suitable for human eyes / tail -f.
 *   - 'json': pretty-printed JSON object with metadata + events array;
 *     suitable for `jq` / downstream tooling.
 *
 * Optional filters: eventKinds (only include these kinds) and
 * sinceTimestamp (only include events >= this epoch ms). Filters
 * apply AFTER readAuditLog returns, so `totalEvents` reflects
 * raw count and `events.length` reflects post-filter count.
 *
 * The function is DEFENSIVE: missing file -> empty result
 * (relying on D-90's ENOENT handling); filter results in zero
 * events -> empty dump; no throws on empty input.
 */

import { readAuditLog } from './audit-log-reader.js';
import type { AuditEvent, AuditEventKind } from './audit-log.js';

export type AuditDumpFormat = 'text' | 'json';

export interface AuditDumpOptions {
  /** Optional: only include events with these kinds. */
  readonly eventKinds?: readonly AuditEventKind[];
  /** Optional: only include events with timestamp >= this (epoch ms). */
  readonly sinceTimestamp?: number;
  /** Output format; default 'text'. */
  readonly format?: AuditDumpFormat;
}

export interface AuditDumpResult {
  readonly filePath: string;
  readonly format: AuditDumpFormat;
  readonly totalEvents: number;
  /** Events actually rendered (after filters applied). */
  readonly events: readonly AuditEvent[];
  /** The rendered output (text or JSON string). */
  readonly dump: string;
  /** Convenience: true iff `events` is empty. */
  readonly isEmpty: boolean;
}

function applyFilters(
  events: readonly AuditEvent[],
  options: AuditDumpOptions,
): AuditEvent[] {
  let filtered: AuditEvent[] = events.slice();
  if (options.eventKinds !== undefined) {
    const allowed: Partial<Record<AuditEventKind, true>> = {};
    for (const k of options.eventKinds) {
      allowed[k] = true;
    }
    filtered = filtered.filter((e) => allowed[e.kind] === true);
  }
  if (options.sinceTimestamp !== undefined) {
    const threshold = options.sinceTimestamp;
    filtered = filtered.filter((e) => e.timestamp >= threshold);
  }
  // Sort by timestamp ascending so text output is chronological.
  filtered.sort((a, b) => a.timestamp - b.timestamp);
  return filtered;
}

function renderText(filePath: string, events: readonly AuditEvent[]): string {
  const lines: string[] = [];
  lines.push('=== AuditLog dump ===');
  lines.push(`File: ${filePath}`);
  lines.push(`Format: text`);
  lines.push(`Total events: ${events.length}`);
  if (events.length === 0) {
    lines.push('');
    lines.push('(no events)');
    lines.push('=== end ===');
    return lines.join('\n');
  }
  const pad = String(events.length).length;
  for (let i = 0; i < events.length; i++) {
    const ev = events[i]!;
    const ts = new Date(ev.timestamp).toISOString();
    const idx = String(i + 1).padStart(pad, '0');
    const payload = JSON.stringify(ev.payload);
    lines.push(`[${idx}] ${ts}  ${ev.kind}  ${payload}`);
  }
  lines.push('=== end ===');
  return lines.join('\n');
}

function renderJson(
  filePath: string,
  totalEvents: number,
  events: readonly AuditEvent[],
): string {
  return JSON.stringify(
    { filePath, format: 'json', totalEvents, events },
    null,
    2,
  );
}

/**
 * Render a persisted audit log as text or JSON. Reads via D-90
 * readAuditLog and applies optional filters.
 */
export async function dumpAuditLog(
  filePath: string,
  options: AuditDumpOptions = {},
): Promise<AuditDumpResult> {
  const format: AuditDumpFormat = options.format ?? 'text';
  const raw = await readAuditLog(filePath);
  const totalEvents = raw.length;
  const events = applyFilters(raw, options);
  const dump =
    format === 'json'
      ? renderJson(filePath, totalEvents, events)
      : renderText(filePath, events);
  return {
    filePath,
    format,
    totalEvents,
    events,
    dump,
    isEmpty: events.length === 0,
  };
}
```
