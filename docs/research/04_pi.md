## 3. 关键技术细节

### 3.1 Extension API 详解

#### 3.1.1 事件总线底座

`packages/coding-agent/src/core/event-bus.ts:1-33`：

```typescript
// 仅 33 行；本质是 Node EventEmitter 的轻量封装
import { EventEmitter } from "node:events";

export interface EventBus {
    emit(channel: string, data: unknown): void;
    on(channel: string, handler: (data: unknown) => void): () => void;
}

export function createEventBus(): EventBusController {
    const emitter = new EventEmitter();
    return {
        emit: (channel, data) => { emitter.emit(channel, data); },
        on: (channel, handler) => {
            const safeHandler = async (data: unknown) => {
                try { await handler(data); }
                catch (err) { console.error(`Event handler error (${channel}):`, err); }
            };
            emitter.on(channel, safeHandler);
            return () => emitter.off(channel, safeHandler);
        },
        clear: () => { emitter.removeAllListeners(); },
    };
}
```

关键点：**handler 自动包裹 `try/catch`**——单个 extension 抛错不会影响其他 extension 的处理。这是 deepwhale 必须照搬的设计（避免"一个插件炸了导致整个 agent 死掉"）。

#### 3.1.2 `defineTool` 的本质：纯类型守卫，零运行时

`packages/coding-agent/src/core/extensions/types.ts:491-495`：

```typescript
export function defineTool<TParams extends TSchema, TDetails = unknown, TState = any>(
    tool: ToolDefinition<TParams, TDetails, TState>,
): ToolDefinition<TParams, TDetails, TState> & AnyToolDefinition {
    return tool as ToolDefinition<TParams, TDetails, TState> & AnyToolDefinition;
}
```

完整 `ToolDefinition`（types.ts:433-468）：

```typescript
export interface ToolDefinition<TParams extends TSchema = TSchema, TDetails = unknown, TState = any> {
    name: string;          // LLM 调用的工具名
    label: string;         // UI 显示
    description: string;   // LLM 看到的描述
    promptSnippet?: string;         // 一句话描述，注入 system prompt "Available tools" 节
    promptGuidelines?: string[];    // 注入 system prompt "Guidelines" 节
    parameters: TParams;            // TypeBox schema
    renderShell?: "default" | "self";
    prepareArguments?: (args: unknown) => Static<TParams>;
    executionMode?: ToolExecutionMode;   // "sequential" | "parallel"
    execute(
        toolCallId: string,
        params: Static<TParams>,
        signal: AbortSignal | undefined,
        onUpdate: AgentToolUpdateCallback<TDetails> | undefined,
        ctx: ExtensionContext,         // ← 关键：执行时拿到 ctx
    ): Promise<AgentToolResult<TDetails>>;
    renderCall?: (args, theme, context) => Component;
    renderResult?: (...) => Component;
}
```

> **`defineTool` 在运行时不做任何事**，仅是 TypeScript 编译期的类型推断守卫（types.ts:484-490 注释："Preserve parameter inference for standalone tool definitions"）。这意味着 deepwhale 可以**只靠泛型 + `as` 强转**就实现它——避免过度设计。

#### 3.1.3 扩展入口装载：`jiti` 动态 import

`packages/coding-agent/src/core/extensions/loader.ts:331-343`：

```typescript
async function loadExtensionModule(extensionPath: string) {
    const jiti = createJiti(import.meta.url, {
        moduleCache: false,
        // Bun binary: use virtualModules for bundled packages
        // Node.js/dev: use aliases to resolve to node_modules paths
        ...(isBunBinary
            ? { virtualModules: VIRTUAL_MODULES, tryNative: false }
            : { alias: getAliases() }),
    });
    const module = await jiti.import(extensionPath, { default: true });
    const factory = module as ExtensionFactory;
    return typeof factory !== "function" ? undefined : factory;
}
```

- 用 `jiti` 让 extension 可以**直接写 TypeScript**而不需要预先编译。
- 区分 Bun 编译产物（用 `virtualModules`）和 Node 开发（用 `alias`），分别走两套模块解析路径。
- 装载流程（loader.ts:368-391）：解析路径 → jiti 加载 → 拿到 factory → `createExtensionAPI()` → 调 `factory(api)` 让 extension 注册 handler / tool / command。

#### 3.1.4 扩展 discovery 规则

`packages/coding-agent/src/core/extensions/loader.ts:515-547`：

```typescript
/**
 * Discovery rules:
 * 1. Direct files: extensions/*.ts or *.js → load
 * 2. Subdirectory with index: extensions/*/index.ts or index.js → load
 * 3. Subdirectory with package.json: extensions/*/package.json with "pi" field → load what it declares
 *
 * No recursion beyond one level. Complex packages must use package.json manifest.
 */
```

**关键规则：只扫描一级，不递归**——复杂包必须用 `package.json` 的 `"pi"` 字段声明入口。deepwhale 应照搬这一约束，避免无限递归扫到 `node_modules`。

`pi` manifest 字段（loader.ts:440-458）：

```typescript
interface PiManifest {
    extensions?: string[];
    themes?: string[];
    skills?: string[];
    prompts?: string[];
}
function readPiManifest(packageJsonPath: string): PiManifest | null {
    try {
        const content = fs.readFileSync(packageJsonPath, "utf-8");
        const pkg = JSON.parse(content);
        if (pkg.pi && typeof pkg.pi === "object") {
            return pkg.pi as PiManifest;
        }
        return null;
    } catch { return null; }
}
```

`pi` 字段是 deepwhale 装三方扩展的"标准接入点"——package.json 里写 `"pi": {"extensions": ["./dist/index.js"], "skills": ["./SKILL.md"]}` 即可。

> - `tool.after` → `tool_result`
> - `message.receive` → `message_end`（注意：pi 的命名是"消息结束"，不是"接收"，因为是流式场景）

#### 3.1.7 Extension 之间通讯：`pi.events`

#### 3.3.1 三种 source 形态

#### 3.3.2 解析算法（package-manager.ts:1380-1403）

```typescript
private parseSource(source: string): ParsedSource {
    if (source.startsWith("npm:")) {
        const spec = source.slice("npm:".length).trim();
        const { name, version } = this.parseNpmSpec(spec);
        return {
            type: "npm",
            spec,
            name,
            pinned: Boolean(version),
        };
    }

    if (isLocalPath(source)) {
        return { type: "local", path: source };
    }

    // Try parsing as git URL
    const gitParsed = parseGitUrl(source);
    if (gitParsed) {
        return gitParsed;
    }

    return { type: "local", path: source };
}
```

> **deepwhale 借鉴点**：
> - `npm:` 前缀是**显式命名空间**。
> - 否则按"本地路径 → git URL → 兜底本地"顺序判断。
> - `parseGitUrl()` 在 `utils/git.ts` 中实现，用 `hosted-git-info` 包解析 GitHub / GitLab / Bitbucket 短链，并支持 `git@host:path@ref` SSH 形式和 `host/path@ref` 短形式（`splitRef`，utils/git.ts:9-78）。

#### 3.3.3 npm 名称 / 版本分离（package-manager.ts:1659-1667）

```typescript
private parseNpmSpec(spec: string): { name: string; version?: string } {
    const match = spec.match(/^(@?[^@]+(?:\/[^@]+)?)(?:@(.+))?$/);
    if (!match) {
        return { name: spec };
    }
    const name = match[1] ?? spec;
    const version = match[2];
    return { name, version };
}
```

> 关键正则 `/^(@?[^@]+(?:\/[^@]+)?)(?:@(.+))?$/` 拆解：
> - `(@?)` —— 可选的 `@`（scoped 包标志）
> - `([^@]+(?:\/[^@]+)?)` —— 第一段是包名（可包含 `/`），scoped 时为 `name/...`
> - `(?:@(.+))?` —— 后面可选的 `@version`（含 scope 的 tag 也匹配）
> 
> **deepwhale 提示**：注意坑——这个正则把 `@org/pkg@1.2.3` 解析为 `name=@org/pkg`, `version=1.2.3`；但对 git ref（如 `git:host/repo@v1#sub=path`）不走这条路径。

#### 3.3.4 安装时调用的 npm / pnpm / bun 命令

```typescript
private getNpmCommand(): { command: string; args: string[] } {
    const configuredCommand = this.settingsManager.getNpmCommand();
    if (!configuredCommand || configuredCommand.length === 0) {
        return { command: "npm", args: [] };
    }
    const [command, ...args] = configuredCommand;
    if (!command) {
        throw new Error("Invalid npmCommand: first array entry must be a non-empty command");
    }
    return { command, args };
}

private getPackageManagerName(): string {
    const npmCommand = this.getNpmCommand();
    const commandParts = [npmCommand.command, ...npmCommand.args];
    const separatorIndex = commandParts.lastIndexOf("--");
    const packageManagerCommand = separatorIndex >= 0 ? commandParts[separatorIndex + 1] : npmCommand.command;
    return packageManagerCommand ? basename(packageManagerCommand).replace(/\.(cmd|exe)$/i, "") : "";
}
```

#### 3.3.5 Scope（user / project / temporary）

`package-manager.ts:116`：`type SourceScope = "user" | "project" | "temporary";`

#### 3.3.6 资源优先级（package-manager.ts:161-177）

```typescript
function resourcePrecedenceRank(m: PathMetadata): number {
    if (m.origin === "package") return 4;
    const scopeBase = m.scope === "project" ? 0 : 2;
    return scopeBase + (m.source === "local" ? 0 : 1);
}
/**
 * Precedence (highest to lowest):
 *   0  project + settings entry (source: "local", scope: "project")
 *   1  project + auto-discovered (source: "auto", scope: "project")
 *   2  user + settings entry (source: "local", scope: "user")
 *   3  user + auto-discovered (source: "auto", scope: "user")
 *   4  package resource (origin: "package")
```

### 3.6 事件流闭环：典型 user prompt 走完一遍

1. **TUI 输入** → `InteractiveMode` → `session.prompt(text, {images})`。
2. `AgentSession._sendPrompt` → 调 `ExtensionRunner.emit("before_agent_start", {prompt, images})`（runner.ts:937-983）。
   - extension 可返回 `{ systemPrompt, message }`（types.ts:1018-1022）。
3. `ExtensionRunner.emit("turn_start")`。
4. `ExtensionRunner.emit("message_start")`。
5. 调 `agent-core` 的 `agentLoop`：`streamFn(model, context, options)` → `pi-ai` 真正发 HTTP 请求。
   - 之前 `emit("before_provider_request", payload)`。
6. 流式 chunks：每个 delta → `emit("message_update", {message, delta})`。
7. 模型想调工具 → `agent-core` 返回 `toolCall`，触发 `emit("tool_call", {toolName, input})`。
   - extension 返回 `{block: true}` → 工具被阻止，返回 `reason` 给 LLM。
   - 否则执行工具：内置工具走 pi 自带；extension 工具走 `wrapRegisteredTool()`（wrapper.ts:17-19）。
8. 工具返回 → `emit("tool_result", {toolName, content, isError})`。
   - extension 可改写 `content` 或追加 `isError: true`。
9. 跳回第 5 步（继续 LLM 循环）或退出（model 自决不再 tool call）。
10. `emit("message_end", {message})` —— extension 可替换 `message`。
11. `emit("turn_end")`、`emit("agent_end")`。
12. `SessionManager.appendEntry(messageEntry)` → 立即 JSONL 落盘。

> 这条事件链让 deepwhale 可以在**几乎任何节点**插入自定义逻辑：消息构造前 / LLM 真正发请求前 / 工具调用前 / 工具返回后 / 消息结束时 / session 边界。

### 5.1 抄 1 不抄 1 的注意事项

- **不要抄 `pi-mom` 整套**——它是历史包袱（Slack bot 个人化产品，issue #412 仍在 rewrite），v0.78 已不放在主 monorepo。
- **不要抄 `pi-tui` native prebuild**——用 Rust 写 win32 / darwin 平台层，deepwhale 如果用 Python + textual/ratatui 可以避免。
- **不要抄 bun binary 编译管线**——deepwhale 用 PyOxidizer / Nuitka 路径完全不同。
- **必须抄的 3 件套**：
  1. **ExtensionEvent 命名**（仅改前缀）：`whale.session_start` / `whale.tool_call` / `whale.tool_result` / `whale.session_shutdown` / `whale.message_end` / `whale.session_before_compact`。
  2. **defineTool 的"零运行时"哲学**——避免过度设计。
  3. **JSONL append-only session DAG**——崩溃安全 + 可 fork / tree navigate。
- **deepwhale 自己的命名建议**：
  - 顶层工具叫 `whale`（命令）+ `deepwhale`（库名）。
  - 内部包名：`@deepwhale/llm` / `@deepwhale/agent-core` / `@deepwhale/tui` / `@deepwhale/coding-agent`。
  - 资源命名空间前缀：`whale:`（npm 命名空间解析用）。

## 7. 一句话总结

pi v0.78.0 = 一个**严格分层的 4 包 monorepo**（`pi-ai` / `pi-agent-core` / `pi-tui` / `pi-coding-agent`），核心创新在 `coding-agent` 内的 **ExtensionEvent 联合类型 + `defineTool` 零运行时设计 + JSONL append-only DAG session + `npm:` 命名空间的 PackageManager**。deepwhale 借鉴时**复制架构、复制事件名（改前缀）、复制设计哲学，避开 4 个高危坑**（v0.75.4 跨包类型回归、extension tool 重名 abort、rebindSession 覆盖 tool、TUI 弹窗阻塞快捷键），就能在 3 个月内搭出第一版对标架构。
