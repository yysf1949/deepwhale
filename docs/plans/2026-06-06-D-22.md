## 范围 (3 sub-task, 1 commit cluster)

### D-22.1 命令历史持久化
- **文件**: `~/.deepwhale/tui-history` (JSONL, 每行 1 条 raw line)
- **上限**: 1000 条 (LRU 截断, 防无限增长)
- **加载**: 启动时 readFile, parse JSONL, `rl.history = loaded.reverse()` (readline 期望最新在尾部)
- **保存**: 每次新 prompt (非空 / 非内建命令) append, 加 `\n` 写入 (atomic fsync 避免 crash 留半行)
- **不动**: REPL / print mode 仍无历史 (TUI-only feature)
- **风险**: 0 (单文件, fs API 简单)

### D-22.2 流式 token spinner
- **帧**: 5 帧 `[⠋ ⠙ ⠹ ⠸ ⠼]`, 80ms / 帧
- **机制**: 走 `\r` carriage return + `\x1b[K` clear-to-end-of-line 覆盖
- **触发**: assistant stream 第一块 onChunk 触发, 内容为空 (delta.content 还没到) 时仍转
- **停止**: turn 完成 / onError / turn abort / user 按 Ctrl+C
- **不动**: 状态栏 + header + 横线 (D-21.2) 仍照常输出
- **风险**: Windows 终端对 `\r` 支持: 走 `process.stdout.cursorTo(0)` + `clearLine(1)` 兜底

### D-22.3 Multi-line input
- **机制**: readline `terminal: true` (从 false 改), 用 `\n` 触发换行 + `\` 续行
  - `\` 结尾 → 续行模式, 不喂给 turn, 下次 `line` 触发时合并
  - `\\n` (escaped) → 实际 `\n` 字符
  - 空 `\` → 取消续行, 提交空 prompt
- **冲突修法**: readline terminal mode 自带 SIGINT 处理, **跟 D-19 P2-Ctrl+C onSigint 撞**
  - 修法: 移除 `terminal: true` 之前设的 `process.on('SIGINT', onSigint)`, 让 readline 自己的 SIGINT 转发给 onSigint
  - 备选: 仍 `process.on('SIGINT', onSigint)`, **不**让 readline 接管 (用 `rl.removeAllListeners('SIGINT')` 强制剥离)
- **不动**: prompt / confirm / session writer 路径
- **风险**: 中 — 跟 SIGINT 链 1 处冲突, 已设计修法
