# D-23.2: TUI 语法高亮 (onChunk 染色)

**Sprint**: 1c-revive-2 (2026-06-06)
**范围**: 用户拍板 A + B + D 全要. D-23.2 语法高亮, 仅染 onChunk assistant stream.
**Ship Ritual**: 1 plan + 1 commit + 1 GitHub Release v1.0.8

## 拍板

- **仅染 onChunk assistant stream** (不染 readline input, 染 cursor 会乱)
- **4 类** (3 染 + 1 不染):
  1. **tool name** (BashTool / ReadFileTool / WriteFileTool / EditTool / GlobTool / GrepTool / ListDirectoryTool / FileReadTool / FileWriteTool) → `toolName` role (跟 status bar 复用)
  2. **number** (整数 / 小数 / 百分号 / 倍数 / 货币 / `+` 后缀 e.g. `100+`) → `success` role
  3. **path** (`./rel` / `/abs` / `node_modules/...`) → `model` role
  4. 其它 → 原样 (不染)
- **优先级**: tool > path > number (用 "先标记后还原" 算法, 区间去重)
- **非 TTY 退化**: 跟 `colorize` 一致, 返原文 (CI / 管道 log 不带 ANSI)
- **forceColor 测试 hook**: 高亮函数第 3 参, 给 unit test 验染色字节用 (生产路径不传)

## 改动

### tui.ts (1 文件, ~80 行)

1. **3 个 regex const** (模块级):
   - `TOOL_NAME_RE`: 9 工具白名单 (跟 `registry.createDefaultRegistry` 同步)
   - `NUMBER_RE`: 数字 (含 `+` 后缀)
   - `PATH_RE`: 文件路径 (`./*.ts` / `/abs/path`)

2. **新 helper**:
   - `export function highlightChunk(text, theme, forceColor?)` — "先标记后还原" 算法
   - 输入: 1 个 chunk 文本 fragment
   - 输出: 染色后文本 (含 ANSI escape) 或原 text (非 TTY)
   - 流程:
     1. 扫 3 遍正则, 收集 `[start, end, role]` 区间 (按优先级 tool > path > number)
     2. 区间去重 (后扫到的覆盖先扫到的)
     3. 按 start 排序, 拼回 (区间内 `colorize(role)`, 区间外原文)
     4. 空字符串 / 无匹配: 返原文

3. **改 2 处 `onChunk` 调用**:
   - `runToolLoop` 路径: `out.write(highlightChunk(chunk.content, theme))`
   - `--no-tool-loop` 路径: `out.write(highlightChunk(chunk.delta.content, theme))`

### tui-smoke.test.ts (1 文件, +9 test)

1. **highlightChunk 8 sub-test (describe 嵌套)**:
   - 非 TTY 默认 → 原文 (退化)
   - isTTY=true 工具名 → toolName role (magenta+bold)
   - isTTY=true 数字 (100+, 2.5x, 50%) → success role (green)
   - isTTY=true 路径 (./rel, /abs) → model role (cyan+bold)
   - isTTY=true 混合 4 类按优先级染色
   - 空字符串 → 空字符串 (no throw)
   - 无匹配 → 原文
   - isTTY=true solarized theme → model 变 blue (theme 切换验证)

2. **D-23.2 端到端**:
   - 跑 TUI, mock LLM 返 4 类混合 content, 验 stdout 含原文 (非 TTY 退化)

3. **test 文件头加 `/* eslint-disable no-control-regex */`** (测 ANSI 必含 `\x1b`, bypass no-control-regex)

## 范围 & 风险

**改动**: 1 commit, 2 文件 +160/-3
- packages/coding-agent/src/modes/tui.ts (+90/-0, 加 3 regex + 1 helper + 改 2 onChunk)
- packages/coding-agent/test/modes/tui-smoke.test.ts (+75/-3, +9 test + file-level eslint disable)

**风险评估**:
- chunk content 一次 fragment 长 50 chars 量级, 正则贪婪 OK (跟 Fish / Starship prompt 同形态)
- 区间去重算法: 复杂度 O(n log n) (排序), text 长 < 10K 时远 < 1ms
- tool 白名单 9 个: 不命中 tool name 的 content 仍可命中 path / number (没漏)
- `+` 后缀 regex: `100+` 命中 `100+` (含 +), 不会跟 `+` 字符冲突 (regex `\d+(?:\+|...)?` 走非贪婪)
- 非 TTY 路径: 跟 colorize 同 `isTty()`, CI/管道 log 干净
