**用户拍板** (2026-06-07): D-29 sub-plan, 1 commit cluster

## 1. 目标

把 `sanitizeReason(input)` 从"长度截断 + 换行折叠 + 去 NUL"扩展为"长度截断 + 换行折叠 + 去 NUL + **3 类 secret redact**"。**0 改 public 签名** (单文件, 6 调用点兼容)。

### 1.1 3 类 secret 正则 (跟行业惯例对齐, 测覆盖)

| 形式 | 正则 | 例子 (redact 后) |
|---|---|---|
| `sk-...` 形式 (OpenAI / DeepSeek / Anthropic 风格) | `/\bsk-[A-Za-z0-9_-]{16,}\b/g` | `sk-***REDACTED***` |
| `Bearer ...` 形式 (OAuth/JWT) | `/Bearer\s+[A-Za-z0-9._\-+/=]{16,}/gi` | `Bearer ***REDACTED***` |
| `key=...` / `KEY=...` 形式 (env 注入) | `/\b(key|KEY|token|TOKEN|secret|SECRET)=([A-Za-z0-9._\-+/=]{8,})/g` | `key=***REDACTED***` |

### 1.2 顺序 (在原有 3 步前)

**当前 sanitizeReason 流程**:
1. 折叠换行
2. 去 NUL
3. 长度截断

**D-29.2 流程** (插入 step 0):
0. **3 类 secret redact** (新增, 顺序: 先 redact 再折叠, 因为 redact 的 key= 会含 `=` 后值, 折叠不会影响)
1. 折叠换行 (现有)
2. 去 NUL (现有)
3. 长度截断 (现有)

### 1.3 Redact marker 选型

- **marker**: `***REDACTED***` (15 字符)
- 不用 `[REDACTED]` 因为含方括号, 跟 policy_decision 事件 reason 字段的"双引号转义"链可能冲突
- 不用 `****` 因为会跟 truncate marker `…[truncated]` 视觉混淆
- 不用 `*` 单字符因为太短, 搜索日志时容易撞正常字符

```ts
/**
 * Sprint 1c-revive-5-D-29.2 (2026-06-07)
 *
 * D-13 (commit 1d414cc) MVP 拍板: 长度上限 200 + 换行折叠 + 去 NUL. 真正的
 * secret detection 留给 D-15.
 * D-15 未补, 留 v1.0 隐患. D-29.2 补:
 *   1. sk-... 形式 (OpenAI/DeepSeek/Anthropic API key 风格)
 *   2. Bearer ... 形式 (OAuth/JWT bearer token)
 *   3. key=... / token=... / secret=... 形式 (env 注入风格)
 *
 * 拍板 (D-29.2): 顺序是 secret redact 先于长度截断, 避免先截断再 redact
 * 漏掉截断边界的 secret. marker 用 ***REDACTED*** 15 字符, 不用 [REDACTED]
 * 是为避开 reason 字段的 quote-escape 链.
 *
 * 拍板红线: 0 改 sanitizeReason 签名, 6 调用点 (tool-loop.ts:318, 352, 359,
 * 381, 405 + 1 链) 全部向后兼容.
 */

const MAX_REASON_LEN = 200;
const TRUNCATED_MARKER = '…[truncated]';
const REDACTED_MARKER = '***REDACTED***';

// 3 类 secret 正则 (Sprint 1c-revive-5-D-29.2 拍板):
// 1. sk- 开头 16+ 字符 (OpenAI/DeepSeek/Anthropic 风格)
const SK_PATTERN = /\bsk-[A-Za-z0-9_-]{16,}\b/g;
// 2. Bearer 头 + 16+ 字符 (OAuth/JWT)
const BEARER_PATTERN = /Bearer\s+[A-Za-z0-9._\-+/=]{16,}/gi;
// 3. key/token/secret= + 8+ 字符 (env 注入)
const KEYVALUE_PATTERN = /\b(key|KEY|token|TOKEN|secret|SECRET)=([A-Za-z0-9._\-+/=]{8,})/g;

function redactSecrets(s: string): string {
  return s
    .replace(SK_PATTERN, REDACTED_MARKER)
    .replace(BEARER_PATTERN, `Bearer ${REDACTED_MARKER}`)
    .replace(KEYVALUE_PATTERN, (_, k) => `${k}=${REDACTED_MARKER}`);
}

export function sanitizeReason(reason: string): string {
  // 0. secret redact (D-29.2 新增) — 先于长度截断, 避免截断边界漏 secret
  let r = redactSecrets(reason);
  // 1. 折叠换行
  r = r.replace(/\r?\n/g, ' / ');
  // 2. 去 NUL
  // eslint-disable-next-line no-control-regex
  r = r.replace(/\u0000/g, '');
  // 3. 长度上限 + 标 truncated
  if (r.length > MAX_REASON_LEN) {
    r = r.slice(0, MAX_REASON_LEN - TRUNCATED_MARKER.length) + TRUNCATED_MARKER;
  }
  return r;
}
```

**sanitize-reason.test.ts 改动** (8 it → 14 it, +6 new):

```ts
// 保留 8 现有 it (D-13 MVP 覆盖), 加 6 新 it:

it('sk-... 形式 secret redact (D-29.2)', () => {
  expect(sanitizeReason('api key: sk-1234567890abcdefghij')).toBe(
    'api key: ***REDACTED***'
  );
});

it('sk- 短于 16 字符不 redact (D-29.2)', () => {
  expect(sanitizeReason('sk-short')).toBe('sk-short'); // 太短不是真 key
});

it('Bearer ... 形式 secret redact (D-29.2)', () => {
  expect(sanitizeReason('header: Bearer abc123def456ghi789jkl012mno')).toBe(
    'header: Bearer ***REDACTED***'
  );
});

it('key=value / token=value / secret=value 形式 redact (D-29.2)', () => {
  expect(sanitizeReason('env DEPLOY_KEY=prod-secret-12345')).toBe(
    'env DEPLOY_KEY=***REDACTED***'
  );
  expect(sanitizeReason('oauth TOKEN=eyJhbGciOiJIUzI1NiJ9.payload')).toBe(
    'oauth TOKEN=***REDACTED***'
  );
});

it('secret redact 在长度截断前 (D-29.2 顺序)', () => {
  const long = 'a'.repeat(180) + ' sk-1234567890abcdefghij ' + 'b'.repeat(50);
  const r = sanitizeReason(long);
  // redact 后长度变化, 但不能漏 secret
  expect(r).toContain('***REDACTED***');
  expect(r).not.toContain('sk-1234567890abcdefghij');
});

it('顺序: 先 redact 再折叠 (D-29.2)', () => {
  // reason 含 key=foo 跨换行, 折叠不应破坏 redact
  const r = sanitizeReason('key=mysecret123\nnext line');
  expect(r).toBe('key=***REDACTED*** / next line');
  expect(r).not.toContain('mysecret123');
});
```
