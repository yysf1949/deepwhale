## 任务清单

### Task 1: 假 key 触发 Windows/libuv assertion 崩溃 (P2 稳定性债 P1 修复)

**根因**:
- DeepSeek 返 401 + JSON error body, body 是 `ReadableStream`
- 之前 `throwOnHttpError` 调 `await res.text()`, 等于 `reader.read()` 读全部 stream
- 跟 streamChat 路径里如果已有 reader 撞 libuv 状态机
- Windows + libuv 内部 assertion (`EBADF` / "Invalid argument") 后 crash 进程

**修法** (deepseek-client.ts:405-425):
- 改 `throwOnHttpError`: 先 `body.getReader()` → 1 个 chunk read (200 chars) → `reader.cancel()` 干净关闭
- 加 `body.locked` 检查: 已锁时跳过 read, 走 message-only, 不死等
- swallow network 异常 (我们只想拿 status + 短 snippet)

**Tests** (+3 unit test, deepseek-client.test.ts):
- 401 + JSON error body → LLMAuthError 含 status + snippet (不 crash stream)
- 401 + body=null (Node <18 fetch 边缘) → LLMAuthError 不死等
- 401 + body.locked (reader 已被占用) → LLMAuthError 不死等

**验证**:
- `pnpm test` 538 passed | 20 skipped (从 469 → 538, +69 tests, 全过)
- `pnpm lint` 0 警告
- `tsc -b` 0 错
