## 4. FAIL 测 root cause 分析

**FAIL 测**: `policy/sanitize-reason.sanitizeReason > secret redact 在长度截断前 (D-29.2 顺序)` (L83-91)

**输入**:
```
'a'.repeat(180) + ' ' + sk-abc...4567 (20 chars) + ' ' + 'b'.repeat(50)
= 252 字符
```

**sanitizeReason 流程** (按 plan §1.2 顺序: redact → 折叠 → 去 NUL → truncate):
1. redact: sk-abc...4567 → ***REDACTED***, 长度 252 → 247 字符
2. 折叠: 247 字符 (无 \n)
3. 去 NUL: 247 字符
4. truncate: 247 > 200, 切前 187 字符 + '…[truncated]' = 200 字符

**实际输出**:
```
'aaa...aaa ***REDA…[truncated]'
```

**断言期望**:
```js
expect(r).toContain('***REDACTED***')  // ❌ FAIL — r 含 '***REDA' 但不含 '***REDACTED***'
```

**Root cause**:
- `***REDACTED***` 长度 = 15 字符
- truncate 在 200 字符边界, 切在 `'…[truncated]'` 13 字符后
- 187 字符位置正好把 `'***REDACTED***'` 切在第 8 字符 (`***REDA`) 处, 剩 `'CTED***'` 被 truncate marker 覆盖
- **矛盾**: plan §1.2 拍板"redact 先, truncate 后" + plan §1.3 拍板"marker 15 字符" + plan §1.2 拍板"长度上限 200" 三者不兼容, truncate 必然切 marker
