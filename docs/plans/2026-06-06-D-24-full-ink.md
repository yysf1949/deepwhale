## 1. 范围

### 1.1 主任务 (A1+A3) — 全量 Ink 装路径

**核心**: TUI mode 从 `node:readline + ANSI` 迁移到 Ink 6.x + React 19, 装路径跟 Hermes 完全对齐。

### 1.2 不动 (红线)

- **REPL (`repl.ts`)**: 保持 `node:readline` 不动。理由: REPL 是 `node bin/deepwhale.js` 默认入口, sprint budget + 兼容性。REPL 跟 TUI 是 2 套 UX 模式, 不算"legacy"残留。
- **print / rpc mode**: 不动。
- **`runToolLoop` / `SessionWriter` / `createReplConfirm` / `formatUsageStatus` / `staticToolPolicy`**: 0 改动 (跟 D-20.3 P0-B 红线一致 — TUI 容器换, 业务路径不重写)。

### 1.3 1.0.9 包含 (用户原话 "全量 Ink 就该全量")

D-22 (历史持久化) + D-23.1 (主题) + D-23.2 (语法高亮) **已经 ship 在 readline 容器里**, 1.0.9 把这 3 项的逻辑搬到 Ink 组件树里 (不重写, 只搬容器)。逻辑零改动, 纯迁移。

## 2. 架构 — 跟 Hermes `ui-tui` 对齐

### 2.1 Hermes 经验 (可借鉴)

| 决策 | Hermes 做法 | 我们采用 |
|---|---|---|
| Ink 子包化 | `ui-tui/packages/hermes-ink/` (私有, esbuild bundle) | `packages/tui-ink/` (私有 workspace 包) |
| 依赖作用域 | bundle 内 peerDeps, runtime 0 deps | 同 — bundle 完 `dist/tui.js` 自包含 |
| React 版本 | 19.2.4 | 跟 19.x (Ink 6 要求) |
| Ink 版本 | 6.8.0 | 跟 6.8.x |
| JSX | `automatic` + `jsxImportSource: 'react'` | 同 |
| Target | `node20` | 同 |
| RPC 通信 | Python `tui_gateway` over JSON-RPC stdio | **不采用** — 我们没有 Python backend, 用 direct `runToolLoop` |
| 多行 / 主题 / 高亮 | 已 ship, 不重写 | D-22 / D-23.1 / D-23.2 逻辑搬到 Ink 组件树 |

### 2.3 coding-agent 改动 (1 处)

- **`packages/coding-agent/src/modes/index.ts`**: 不删 `runTuiMode`, 但标记 **legacy**
- **`packages/coding-agent/src/modes/tui.ts`**: 整段保留, 不动 (供 fallback 用)
- **`packages/coding-agent/bin/deepwhale.js`**: 加 dispatch —
  ```js
  if (argv.mode === 'tui') {
    // 1. 解析是否在 npm-install tarball 场景
    // 2. 优先 require @deepwhale/tui-ink/dist/tui.js (bundle)
    // 3. 找不到 (dev / source install) fallback 到 src/modes/tui.ts (legacy)
  }
  ```
  - **不**直接删 `runTuiMode`, 让 npm-install tarball 路径走 bundle 没问题, 但 source install 仍能用 (避免 dev 环境被堵)

## 4. 风险 + 缓解

| 风险 | 概率 | 缓解 |
|---|---|---|
| Ink bundle 太大, tarball 体积涨 | 中 | 验 bundle < 1.5MB (Hermes 同量); pnpm publish 体积记录 |
| React 19 peer dep 跟某 transitive 撞 | 低 | 用 `pnpm.peerDependencyRules.allowedVersions` (跟 Hermes 同样手法) |
| SIGINT 透传在 Ink 容器内行为不同 | 中 | `useInput` 捕获 `ctrl+c`, 跟 D-19 P2 onSigint 链兼容, 单测覆盖 |
| TUI smoke test 979 行 (readline-based) 全部要重写? | **不重写** | 新增 `tui-ink/test/`, 不删 readline 测试; 两条路径并行 |
| Windows `setRawMode` 跟 Ink AlternateScreen 撞 | 中 | 跟 Hermes 一样 `resetTerminalModes()` 启动期清理 |
| bundle 在 tarball 装路径下 require 失败 (路径相对/绝对) | 中 | esbuild `outfile` 用 `__dirname` 相对; `require.resolve` 测 |
| 用户运行 `node bin/deepwhale.js tui` 看不出区别 | 低 | README + help 文案明确 `TUI (Ink)` vs `REPL` |
