## 3. 子模块边界契约

### 3.1 `repl-session.ts` (新)

**职责**:
- `loadSessionOrInit(sessionPath?)` — 路径解析 + loadSession (D-13)
- `openWriter(sessionPath)` — SessionWriter.open
- `closeWriter(writer, exitCode)` — try/finally 收尾
- `accumulateUsage(usage, prev)` — EMA 状态 (repl.ts:733 appendUsageStatus 当前实现)
- `formatUsageStatus(usage, model, locale)` — 货币格式化 (USD/CNY)

**导出**:
```ts
export interface SessionHandle {
  writer: SessionWriter | null;
  sessionPath: string | undefined;
  accumulatedUsage: UsageStats;
}

export async function openSession(opts: { sessionPath?: string }): Promise<SessionHandle>
export async function closeSession(handle: SessionHandle, exitCode: number): Promise<void>
export function accumulateUsage(current: UsageStats, prev: UsageStats): UsageStats
export function formatUsageStatus(usage: UsageStats, model: string, locale: Locale): string
```

### 3.2 `repl-command-router.ts` (新)

**职责**:
- `routeBuiltin(line, ctx)` — 派发 /help /exit /verify /unknown (含 D-19.5p 修法: turn guard 之后派发 + /verify 嵌套 try/finally)
- `handleExit(ctx)` — fast-path, 写 pendingExit 或 finish
- `handleVerify(ctx)` — 调用 runVerify + 写 verification event + 输出
- `handleHelp(ctx)` — output.write 提示
- `handleUnknown(ctx, line)` — output.write 未知命令提示

**导出**:
```ts
export interface RouterContext {
  confirmController: ReplConfirmController;
  out: NodeJS.WritableStream;
  err: NodeJS.WritableStream;
  writer: SessionWriter | null;
  rl: readline.Interface;
  finish: (code: number) => Promise<void>;
  // D-19.5p 闭包状态
  turnInFlightRef: { value: boolean };
  lineQueueRef: { value: string[] };
  pendingExitRef: { value: boolean };
  turnAbortControllerRef: { current: AbortController | null };
}

export type RouteResult = 'consumed' | 'queued' | 'exited';

export async function routeBuiltin(line: string, ctx: RouterContext): Promise<RouteResult>
```

### 3.3 `repl-signal-coordinator.ts` (新)

**职责**:
- `attachSigint(ctx)` — 挂 SIGINT 监听, dismiss in-flight confirm + abort turn
- `detachSigint(ctx)` — finish 入口 process.off (D-19.5 P2 红线)
- `newTurnAbortController(ctx)` — 每次 turn 入口 new AbortController
- `exposeTurnControllerForTest(controller)` — 测试环境 expose (D-19 P2 链路)

**导出**:
```ts
export interface SignalContext {
  confirmController: ReplConfirmController;
  turnAbortControllerRef: { current: AbortController | null };
  onSigintDismissConfirmThenAbort: () => void;
}

export function attachSigint(ctx: SignalContext): void
export function detachSigint(): void  // process.off(SIGINT, onSigint)
export function newTurnAbortController(ctx: SignalContext): AbortController
```

### 3.4 `repl.ts` (重写, ≤ 300L)

**职责**: 仅组装 + 入口函数
- import 3 个新模块 + 已有 repl-confirm
- 闭包内创建 ctx 对象 (lineQueue / pendingExit / turnInFlight / 闭包共享)
- `startRepl(options)` = `await openSession(...)` + `attachSigint(...)` + `await new Promise((resolve) => { rl.on('line', async (line) => { ... }) })`
- `runAgentTurn(...)` = 抽出 useRunToolLoop 桥 (D-28 tui-ink 已 ship 同模式, 复用 if possible)

**导出**:
```ts
export { startRepl, runAgentTurn }  // 签名 0 改
```
