## §2 D-29.1.1 signal-coordinator 抽出 (本 commit scope)

### §2.1 抽什么 (L286-307 + L342-363 + L583-585 SIGINT 相关片段)

```ts
// L342-363: SIGINT handler + process.on
let turnAbortController = new AbortController();
const onSigint = (): void => {
  if (confirmController.hasPending()) {
    confirmController.dismiss();
  }
  if (!turnAbortController.signal.aborted) {
    turnAbortController.abort();
  }
};
process.on('SIGINT', onSigint);

// L286-307: finish() 入口 process.off + clearTimeout
const finish = async (code: number): Promise<void> => {
  if (exiting) return;
  exiting = true;
  if (exitTimer) {
    clearTimeout(exitTimer);
    exitTimer = null;
  }
  process.off('SIGINT', onSigint);
  rl.close();
  // ... writer.close() + out.write(goodbye) + resolve
};

// L510: turn 入口续命 turnAbortController
turnAbortController = new AbortController();

// L583-585: close handler 里 abort
if (turnInFlight && !turnAbortController.signal.aborted) {
  turnAbortController.abort();
}
```

### §2.2 抽后接口 (repl-signal-coordinator.ts)

`createSignalCoordinator(opts)` 返 `ReplSignalCoordinator` controller, 形态跟
`createReplConfirm` 对齐:

```ts
export interface ReplSignalCoordinatorOptions {
  /** SIGINT 触发时先 dismiss 的 confirm controller (D-19 P2-dismiss 顺序). */
  confirmController: ReplConfirmController;
  /** finish() 入口调 (process.off + rl.close 之前). */
  onFinish: () => void;
  /** 测试用 — 注入 process 默认用 node:process. */
  process?: NodeJS.Process;
}

export interface ReplSignalCoordinator {
  /** 当前 in-flight turn 的 AbortSignal (new turn 时 refresh). */
  getSignal: () => AbortSignal;
  /** 续命 controller (turn 入口 new 后调, signal 派给 runAgentTurn). */
  refresh: () => void;
  /** 主动 abort (close 路径 / pendingExit 兜底). */
  abortIfActive: () => void;
  /** finish() 入口调: process.off + 幂等. */
  dispose: () => void;
}
```

### §2.3 抽后 repl.ts 改法 (5 处)

1. **L55** 加 import: `import { createSignalCoordinator } from './repl/repl-signal-coordinator.js';`

3. **L342-363** 整段 (let turnAbortController + onSigint + process.on) 替换为:

6. **L583-585** `if (turnInFlight && !turnAbortController.signal.aborted) { turnAbortController.abort(); }` 改为 `if (turnInFlight) { signalCoordinator.abortIfActive(); }`

## §6 Risks (拍板已知)

- **R-1**: 抽 signal-coord 后 onSigint 闭包持有 confirmController + signalCoordinator
  双向引用, 闭包图略复杂. 可接受: 跟 `repl-confirm.ts` 内部 `pending` 闭包同形态,
  单元隔离 OK.
- **R-2**: dispose() 幂等性靠闭包 `disposed` boolean, 多次调安全. 跟 D-19.5 finish()
  拍板 (off 在 close 之前) 一致.
- **R-3**: 单测不写 — D-19 拍红线 "测 SIGINT 行为走 turnAbortController.abort() 直接调,
  不挂真 process". 端到端 SIGINT 走 repl-shared-stdin 集成测覆盖 (D-19 ship 时已加).
